Domain Name System

Imagine DNS server as a phonebook. In the same way as phone book maps easily recognizable names to the obscure 10-digit numbers a domain name system (DNS) is the Internet’s naming service that maps human-friendly domain names to machine-readable IP addresses. Here are the four key components in a Domain Name System.

  • Name servers: DNS servers that respond to users’ queries are called name servers.
  • Resource records: The DNS database stores domain name to IP address mappings in the form of resource records (RR).
  • Caching: Caching refers to the temporary storage of frequently requested resource records.
  • Hierarchy: DNS name servers are in a hierarchical form. The hierarchical structure allows DNS to be highly scalable.
  • Consistency: DNS compromises on strong consistency to achieve high performance.

There are mainly four types of servers in the DNS hierarchy:

DNS resolver

Resolvers initiate the querying sequence and forward requests to the other DNS name servers.

Root-level name servers

These servers receive requests from local servers.

Top-level domain (TLD) name servers

These servers hold the IP addresses of authoritative name servers.

Authoritative name servers

These are the organization’s DNS name servers that provide the IP addresses of the web or application servers.

Load Balancers

Load balancers improve the scalability availability and performance of software applications.

There are two types of load balancers:

  • Global server load balancing (GSLB): GSLB involves the distribution of traffic load across multiple geographical regions.
  • Local load balancing: This refers to load balancing achieved within a data center.

Load balancers can employ static or dynamic algorithms.

Depending on the requirements load balancing can be performed at the network/transport and application layer of the open systems interconnection (OSI) layers.

Tier-0 and tier-1 LBs

If DNS can be considered as the tier-0 load balancer equal cost multipath (ECMP) routers are the tier-1 LBs.

Tier-2 LBs

The second tier of LBs include layer 4 load balancers.

Tier-3 LBs

Layer 7 LBs provide services at tier 3.

Content Delivery Network (CDN)

Main components of CDN are the origin server edge proxy servers which store the content a distribution system to push/pull the content to the proxy servers a routing system client talks to before zeroing in on a proxy server a scrubber server that resides between the client and proxy and management system to the help the origin servers to optimize the content to be pushed to proxys.

Multi-tier CDN architecture

CDN addresses three important problems: high latency, redundant calculations, and scarce data centre resources.

DNS Redirection

DNS redirection is a technique to direct users to the closest proxy server.

Anycast

Anycast allows different destinations to be reachable through the same IP address.

Client multiplexing

Client multiplexing allows a single server to serve multiple clients simultaneously.

HTTP redirection

HTTP redirection uses HTTP status codes to direct traffic.

Content Consistency

Maintaining content consistency across various nodes is crucial for CDN.

Sequencer

In distributed systems the order in which messages are processed matters. A sequencer is a node that generates unique and ordered sequence numbers. Sequencers help to serialize the access to a shared resource or the communication between nodes.
