Mentor-Mentee Platform Requirements

Overview

This web application will facilitate mentorship by enabling mentors and mentees to manage and track learning roadmaps. Developed using Java, Spring Boot, Thymeleaf, HTML, and CSS, the application will serve two main types of users: Mentors and Mentees. The platform will be responsive and viewable on PCs, tablets, and phones.

1. User Roles & Permissions

1.1 Mentor

Role: Mentors manage the learning roadmap and track mentee progress.

Permissions:

  • Add, update, view, and delete learning roadmap items.
  • Add, update, view, and delete details of their mentees.
  • View progress of all mentees, with the most accomplished displayed at the top.
  • Double-click on a menteeâ€™s details to view their assigned roadmap items and progress.

1.2 Mentee

Role: Mentees track their learning progress and manage assigned roadmap items.

Permissions:

  • Assign items from the learning roadmap to themselves.
  • Update the status of their progress on assigned items.
  • View their assigned roadmap items and track their progress.

2. Functional Requirements

2.1 Mentor Functionalities

2.1.1 Manage Learning Roadmap

Mentors can create new learning roadmap items, which can be books, online trainings, YouTube videos, or podcasts. They can update details such as title, type, description, and other relevant information, and delete items if necessary.

2.1.2 Manage Mentees

Mentors can add new mentees, update their details, and remove them if necessary.

2.1.3 View Mentee Progress

Mentors can view the progress of all their mentees, with the mentees who have made the most progress listed at the top. They can double-click on a mentee's name to see detailed information about their assigned roadmap items and their progress.

2.2 Mentee Functionalities

2.2.1 Assign Roadmap Items

Mentees can select items from the learning roadmap and assign them to themselves. They should be able to specify their current progress on these items.

2.2.2 Update Progress

Mentees can update their progress on assigned roadmap items, including status changes and any relevant notes.

2.2.3 View Assigned Items

Mentees can view a list of all the items they have assigned to themselves, along with their progress and any associated details.

3. Non-Functional Requirements

3.1 Responsiveness

The application must be fully responsive using Bootstrap to ensure it is viewable on PCs, tablets, and mobile devices.

3.2 Security

Implement role-based access control (RBAC) to ensure mentors and mentees can only access functionalities specific to their roles. Secure handling of user data and progress updates using SSL encryption. Data validation on both client-side (using JavaScript) and server-side.

3.3 Performance

The application should handle up to 500 simultaneous users without significant performance degradation. Optimize database queries and page load times for a seamless user experience.

3.4 Scalability

The system architecture should allow for easy scaling to accommodate more users and roadmap items as the platform grows.

4. Technical Stack

  • Backend: Java, Spring Boot
  • Frontend: Thymeleaf, HTML, CSS, Bootstrap
  • Database: MySQL/PostgreSQL/H2 (or any suitable relational database)

5. Development & Testing

5.1 Development

Agile methodology will be followed, with iterative sprints for continuous development and feature integration. A CI/CD pipeline will be established for automated testing and deployment.

5.2 Testing

Please consider writing test cases as optional for now. Unit tests for backend services using JUnit. Frontend testing with Selenium or an equivalent tool. Security testing to ensure the application is secure against common vulnerabilities like SQL injection, XSS, etc. Performance testing to verify that the application meets performance and scalability requirements.

6. Deployment

The application will be deployed only in your local machine. The source code should be added to internal github on a daily basis.

7. Documentation

Comprehensive user and technical documentation will be provided to assist in future maintenance and upgrades.
