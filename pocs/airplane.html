Private Airline Management System Requirements

Overview

This web application will manage the booking and management of aircraft for a private airline company. Developed using Java, Spring Boot, Thymeleaf, HTML, and CSS, the application will provide a web interface for two types of users: Managers and Flying Enthusiasts. The application will handle aircraft inventory, route management, and booking processes. It will be responsive and accessible on PCs, tablets, and mobile devices.

1. User Roles & Permissions

1.1 Manager

Role: Manage aircraft inventory and routes, and oversee booking and return operations.

Permissions:

  • Add, update, view, and delete aircraft details in the inventory.
  • Add, update, view, and delete route details.
  • Manage bookings and returns, including handling aircraft availability based on the elapsed time.

1.2 Flying Enthusiast

Role: Book and return aircraft.

Permissions:

  • View available aircraft and routes.
  • Select aircraft and book it for a specific route.
  • Return the aircraft after the total time has elapsed, making it available for other enthusiasts.

2.1.2 Manage Routes

Managers can add, update, view, and delete route details. Each route is associated with one or two specific aircraft.

Routes Table:

  • Route Name
  • Aircraft(s) Allowed
  • Total Time for Round Trip

2.1.3 Booking and Return Management

Managers can view bookings, handle returns, and ensure aircraft availability based on the elapsed time.

2.2 Flying Enthusiast Functionalities

2.2.1 View Aircraft and Routes

Enthusiasts can view the list of available aircraft and the routes they can fly.

2.2.2 Book Aircraft

Enthusiasts can book an aircraft for a specific route. Once booked, the aircraft is unavailable for other bookings until the total time has elapsed.

2.2.3 Return Aircraft

Enthusiasts must return the aircraft after the total time has elapsed, making it available for other enthusiasts.

3. Non-Functional Requirements

3.1 Responsiveness

The application must be fully responsive using Bootstrap to ensure it is viewable on PCs, tablets, and mobile devices.

3.2 Security

Implement role-based access control (RBAC) to ensure managers and enthusiasts only access functionalities specific to their roles. Secure handling of booking data and aircraft information with SSL encryption. Data validation on both client-side (using JavaScript) and server-side.

3.3 Performance

The application should handle up to 200 simultaneous users without significant performance degradation. Optimize database queries and page load times for a seamless user experience.

3.4 Scalability

The system architecture should allow for easy scaling to accommodate more users, aircraft, and routes as the platform grows.

4. Technical Stack

  • Backend: Java, Spring Boot
  • Frontend: Thymeleaf, HTML, CSS, Bootstrap
  • Database: MySQL/PostgreSQL/H2 (or any suitable relational database)

5. Development & Testing

5.1 Development

Agile methodology will be followed, with iterative sprints for continuous development and feature integration. A CI/CD pipeline will be established for automated testing and deployment.

5.2 Testing

Please consider writing test cases as optional for now. Unit tests for backend services using JUnit. Frontend testing with Selenium or an equivalent tool. Security testing to ensure the application is secure against common vulnerabilities like SQL injection, XSS, etc. Performance testing to verify that the application meets performance and scalability requirements.

6. Deployment

The application will be deployed only in your local machine. The source code should be added to internal github on a daily basis.

7. Documentation

Comprehensive user and technical documentation will be provided to assist in future maintenance and upgrades.
